# Crawl Reviews

## Goal

Populate the `reviews` table with realistic sample reviews generated by a bot user.

## Prerequisites

- PostgreSQL is running.
- Books have already been imported with `import.py`.
- `.env` contains a valid `DATABASE_URL` or the `DB_*` settings needed to build one.
- Dependencies from `pyproject.toml` and `uv.lock` are installed with `uv sync`.

## Inputs

- `books` table
- `reviews` table
- `.env`
- Optional `--limit` argument

## Tools/Scripts

- `execution/crawl_reviews.py`
- Google Books API for optional snippets/descriptions
- Built-in generic review templates when API data is unavailable

## Steps

Run a small batch first:

```powershell
uv run python execution/crawl_reviews.py --limit 5
```

Increase the limit after confirming inserts work:

```powershell
uv run python execution/crawl_reviews.py --limit 50
```

## Behavior

- Creates `bot@bookreviews.local` if it does not already exist.
- Finds books that the bot has not reviewed yet.
- Fetches a snippet or description from Google Books when possible.
- Falls back to generic randomized review text if the API fails.
- Inserts one review per selected book with a random rating from 3 to 5.
- Adds a short randomized delay between API requests.

## Outputs

- New rows in the `reviews` table.
- Console logs showing processed books, generated ratings, and insert results.

## Edge Cases

- If the script prints `No new books to review. Exiting.`, the bot has already reviewed all available books for the requested query.
- If `.env` uses `DB_HOST=db` while the script runs on the host, the script rewrites `@db:` in `DATABASE_URL` to `@localhost:` automatically.
- If Google Books is slow, rate-limited, or missing data, the fallback review templates are used.
- If the database is empty, run `uv run python import.py` first.
